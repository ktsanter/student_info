Help for "Student info" Chrome extension
general
  1. select student
    enter the beginning of a student's first or last name to filter the selection list
  2. student info
    all the information from the data source for the student except for badge items
  3. badges
    information items can be described by badges for specific items. For details, see the badges section below
  4. notes
    add, edit, and delete time-stamped notes
  5. section
    the section in which the student is enrolled
  6. section navigation
    icons are present when a student is in multiple sections.
configuration
The source of the student information must be a Google Sheet and you are welcome to get started by copying this one.

The spreadsheet must follow these guidelines:
  • The first row contains the names of each data field
  • The second row describes the type of data in the field.
    • text - values will display in the "student info" section
    • dontrender - values will not be displayed
    • notes - values will be used in the "notes" section
    • badge_XXX - values will be shown as badges
    • label - the label used at the bottom "course" section
  • All remaining rows contain data for a student
A field for the student's name and for his/her section must be included, e.g. full name (formatted as "last, first"), and section
The type for student's name must be index and the type for the section must be label.
When you open the extension for the first time or when you choose "Configure" from the menu,
you will be prompted to enter the URL for the Google Sheet.

Important:
  • the permissions must be set to "Anyone with the link can view"
  • if a notes field is used the permissions must be set to "Anyone with the link can edit"
  • use the "Get shareable link" option in Drive to get the URL
badges
There are a number of badge types available
  • badge_info, badge_caution, badge_danger - if there is a non-blank value the corresponding badge will be shown:
    and the hover text will show whatever the data value is
  • badge_latewindow - the value must be a date. The date will be compared to the current date and if it's later than or equal to it one of these badges will be shown:
    and the hover text will show either "XXX is late (due nn/nn/nnnn)" or "XXX is due soon : nn/nn/nnnn" where XXX is the column title
  • badge_grade - for values of 9, 10, 11, and 12 these badges will be shown:
    and the hover text will show the grade level
  • badge_iep, badge_504 - if the value is TRUE the relevant badge will be shown:
    and the hover text will say "has IEP" or "has 504"
  • badge_earlygrade - if there is a non-blank value this badge will be one of these, depending on the due date:
    and the hover text will date value and an appropriate message
  • badge_nowelcomeletter - if there is a non-blank value this badge will be shown:
    and the hover text will say "welcome letter not sent yet"
  • badge_multiplementors - if the value is TRUE this badge will be shown:
    and the hover text will say "multiple mentors"
